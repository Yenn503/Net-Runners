## 🔍 Что такое Net-Runner?

Net-Runner — это многоагентный фреймворк для тестирования безопасности, рассчитанный на работу через естественный язык.

Вы подключаете LLM, описываете цель и задачу обычными словами, и Net-Runner запускает весь движок:

- определяет намерение оценки
- создаёт проектный runtime-контур `.netrunner/`
- внедряет scope и контекст workflow в текущую сессию
- направляет работу профильным агентам, когда это нужно
- сохраняет доказательства, память и отчёты по мере выполнения оценки

```text
Вы описываете задачу.
Net-Runner планирует, делегирует, выполняет, запоминает и формирует отчёт.
```

---

## ✨ Почему им пользуются

- **Сначала естественный язык** — не нужно учить команды, чтобы начать
- **Одна встроенная система** — агенты, инструменты, доказательства, память и отчёты идут в одном потоке
- **Профильные агенты** — роли для recon, web, API, network, exploit, AD, retest, evidence и reporting уже подключены
- **Постоянная память** — полезный контекст можно доставать между сессиями через RAG-поиск
- **Сначала доказательства** — находки, шаги выполнения, approvals и отчёты остаются привязанными к одному engagement

## 🚀 Начать отсюда

### 1. Установить и собрать

```bash
bun install
bun run build
```

### 2. Подключить модель

#### `ANTHROPIC_API_KEY`

```bash
export ANTHROPIC_API_KEY="sk-ant-..."
node dist/cli.mjs
```

#### OpenAI

```bash
export OPENAI_API_KEY="sk-..."
export OPENAI_MODEL="gpt-4o"
node dist/cli.mjs
```

#### Google Gemini

```bash
export GEMINI_API_KEY="AIza..."
export GEMINI_MODEL="gemini-2.5-pro"
node dist/cli.mjs
```

#### Ollama

```bash
ollama serve
ollama pull llama3.1:8b
export OPENAI_BASE_URL="http://localhost:11434/v1"
export OPENAI_MODEL="llama3.1:8b"
node dist/cli.mjs
```

#### Любой API, совместимый с OpenAI

```bash
export OPENAI_API_KEY="your-key"
export OPENAI_BASE_URL="https://your-provider.com/v1"
export OPENAI_MODEL="your-model"
node dist/cli.mjs
```

### 3. Говорить естественно

```text
Оцени https://target.example. Начни с разведки, найди основную поверхность атаки, проверь наиболее вероятные проблемы и сохраняй доказательства по ходу работы.
```

Net-Runner распознает цель, запустит engagement, внедрит нужный контекст и начнёт использовать свой agentic runtime.

## ⚙️ Как это работает

```text
Вы
  ↓
Основная LLM-сессия
  ↓
Контекст runtime Net-Runner
  ↓
Специализированные агенты + инструменты + память + доказательства
  ↓
Структурированный результат оценки
```

| Шаг | Что делает Net-Runner |
|-----|------------------------|
| **1. Определяет** | Понимает намерение оценки, тип цели и наиболее вероятный workflow |
| **2. Инициализирует** | Создаёт состояние `.netrunner/` для engagement, если его ещё нет |
| **3. Внедряет** | Добавляет scope, границу воздействия, workflow и базовые skills в текущую сессию |
| **4. Маршрутизирует** | Использует основной runtime и профильных агентов вместе, не заставляя вас вручную управлять командами |
| **5. Защищает** | Применяет внутренние guardrails к разрушительным, persistent или выходящим за scope действиям |
| **6. Записывает** | Сохраняет доказательства, шаги выполнения, находки, ревью, память и отчёты в одной среде |

В обычном режиме вы можете:

- дать цель
- сказать, какой тип оценки запускать
- попросить продолжить, углубить, перепроверить, суммировать или сформировать отчёт
- позволить системе использовать уже доступные среду, инструменты, память и агентов

## 🕵️ Агенты

Net-Runner сохраняет исходный общий agentic-поток и добавляет поверх него специализированные роли безопасности.

| Агент | Что делает |
|:------|:-----------|
| **Engagement Lead** | Оркестрирует оценку, выбирает фазы workflow и распределяет работу |
| **Recon Specialist** | Находит хосты, сервисы, поддомены, технологии и поверхность атаки |
| **Web Testing Specialist** | Проверяет маршруты, параметры, auth-потоки и web-уязвимости |
| **API Testing Specialist** | Проверяет API, схемы, JWT, пути IDOR и переходы состояний |
| **Network Testing Specialist** | Выполняет перечисление сервисов, сетевую проверку и тесты на уровне хоста |
| **Exploit Specialist** | Контролируемо подтверждает реальное воздействие |
| **Privilege Escalation Specialist** | Работает с путями повышения привилегий после первоначального доступа |
| **Lateral Movement Specialist** | Работает с pivot, trust-path и перемещением между хостами |
| **AD Specialist** | Сфокусирован на Active Directory и Kerberos |
| **Retest Specialist** | Повторяет находки и проверяет исправления |
| **Evidence Specialist** | Организует артефакты и трассируемые доказательства |
| **Reporting Specialist** | Превращает доказательства в понятный отчёт |

Базовые runtime-агенты вроде `general-purpose`, `Explore`, `Plan` и `verification` тоже остаются частью системы.

## 🧱 Структура проекта

```text
.netrunner/
├── engagement.json
├── run-state.json
├── evidence/
│   └── ledger.jsonl
├── findings/
├── reports/
├── artifacts/
├── memory/
│   ├── private.md
│   ├── team.md
│   └── agents/
└── instructions/
```

- `engagement.json` — текущий workflow, цели, граница воздействия и ограничения
- `run-state.json` — шаги выполнения и ожидающие ревью
- `evidence/` — append-only журнал доказательств
- `findings/` — структурированные результаты находок
- `reports/` — сгенерированные отчёты
- `artifacts/` — собранные выводы и вспомогательные файлы
- `memory/` — постоянная память оператора, команды и агентов
- `instructions/` — project-scoped runtime-инструкции
